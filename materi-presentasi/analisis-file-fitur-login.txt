ANALISIS FILE FITUR LOGIN
PROJECT: AURA HOTEL WEB

Dokumen ini menjelaskan file dan kode yang membahas fitur login pada project ini. Fokus pembahasan mencakup login, register, forgot password, reset password, callback auth, session, middleware, role-based redirect, serta fondasi database yang mendukung autentikasi.

======================================================================
1. GAMBARAN UMUM STRUKTUR FITUR LOGIN
======================================================================

Secara umum, fitur login di project ini tidak berada dalam satu file saja, melainkan tersebar pada beberapa lapisan:

1. Halaman antarmuka login dan register
2. Action server untuk autentikasi
3. Callback route untuk OAuth dan magic link
4. Helper role, profile, dan redirect
5. Middleware untuk proteksi route
6. Helper session Supabase untuk server dan client
7. Komponen UI yang menampilkan status login
8. Struktur database untuk profile dan role user

Bagian terpenting untuk dipelajari adalah:
- halaman login
- action auth
- callback auth
- helper auth
- middleware

```ts
export function sanitizeInternalRedirect(redirectTo?: string | null) {
  ...
}

export function getPostAuthRedirect(role, redirectTo) {
  const safeRedirect = sanitizeInternalRedirect(redirectTo);

  if (safeRedirect) {
    if (safeRedirect.startsWith("/admin") && !isAdminRole(role)) {
      return "/dashboard";
    }

    return safeRedirect;
  }

  return getRoleHomePath(role);
}
```

Analisis:
Bagian ini sangat penting karena menggabungkan dua hal:
- keamanan redirect
- aturan role-based redirect

8. KESIMPULAN
======================================================================

Jika ditanya file mana saja yang membahas fitur login, maka jawaban intinya adalah:

1. Folder `src/app/login` | File `page.tsx`
   Fungsi: UI login utama

2. Folder `src/app/auth` | File `actions.ts`
   Fungsi: logika login, register, forgot password, reset password

3. Folder `src/app/auth/callback` | File `route.ts`
   Fungsi: callback OAuth dan magic link

4. Folder `src/components` | File `GoogleAuthButton.tsx`
   Fungsi: login Google OAuth

5. Folder `src/lib` | File `auth.ts`
   Fungsi: helper role, profile, redirect, dan auth error

6. Folder `src/utils/supabase` | File `middleware.ts`
   Fungsi: proteksi route dan redirect berbasis session

7. Folder `src/utils/supabase` | File `server.ts` dan `client.ts`
   Fungsi: client Supabase untuk server dan browser

8. Folder `src/app/register`, `src/app/forgot-password`, `src/app/reset-password`
   Fungsi: UI flow autentikasi pendukung

9. Folder root project | File `README.md` dan `supabase_setup.sql`
   Fungsi: dokumentasi dan fondasi database auth

Dengan demikian, fitur login di project ini merupakan gabungan antara UI, server action, callback, middleware, session helper, dan database profile user.
